# @linky/mobile

The Linky Expo app: custom development client (`expo-dev-client`), CNG/prebuild,
Expo Router. Never Expo Go.

## Native projects are generated (CNG)

`ios/` and `android/` are **gitignored** and regenerated by prebuild. All native
configuration lives in `app.config.ts` and Expo config plugins — hand-editing
the generated native projects is forbidden. Any native change (permissions,
entitlements, push, NFC, deep-link schemes) must be expressible as config or a
config plugin.

## Running from a fresh checkout

```sh
pnpm install                 # repo root

cd apps/mobile
npx expo prebuild            # generates ios/ and android/ (add --platform ios|android)
npx expo run:ios             # build + install + launch on an iOS simulator
npx expo run:android         # build + install + launch on an Android emulator
```

`expo run:*` runs prebuild automatically when the native project is missing, so
the explicit prebuild step is optional. After changing `app.config.ts` or
config plugins, run `npx expo prebuild --clean` to regenerate from scratch.

Day-to-day JS development only needs the dev client installed once; after that,
`pnpm start` (Metro) and reloading is enough. Rebuild the native app only when
native dependencies or config change.

## Routes

Expo Router, file-based, under `app/`:

```
app/
  _layout.tsx          root stack
  (tabs)/
    _layout.tsx        tab bar
    index.tsx          Contacts (placeholder)
    wallet.tsx         Wallet (placeholder)
    settings.tsx       Settings (placeholder)
```

Placeholder screens only — features land per the feature map. Styling
(NativeWind + theme tokens) arrives in issue #5; environment profiles
(dev/staging/prod bundle IDs and endpoints) in issue #4.
