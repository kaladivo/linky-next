# @linky/mobile

The Linky Expo app: custom development client (`expo-dev-client`), CNG/prebuild,
Expo Router. Never Expo Go.

## Native projects are generated (CNG)

`ios/` and `android/` are **gitignored** and regenerated by prebuild. All native
configuration lives in `app.config.ts` and Expo config plugins — hand-editing
the generated native projects is forbidden. Any native change (permissions,
entitlements, push, NFC, deep-link schemes) must be expressible as config or a
config plugin.

## Running from a fresh checkout

```sh
pnpm install                 # repo root

cd apps/mobile
npx expo prebuild            # generates ios/ and android/ (add --platform ios|android)
npx expo run:ios             # build + install + launch on an iOS simulator
npx expo run:android         # build + install + launch on an Android emulator
```

`expo run:*` runs prebuild automatically when the native project is missing, so
the explicit prebuild step is optional. After changing `app.config.ts` or
config plugins, run `npx expo prebuild --clean` to regenerate from scratch.

Day-to-day JS development only needs the dev client installed once; after that,
`pnpm start` (Metro) and reloading is enough. Rebuild the native app only when
native dependencies or config change.

## Two-instance verification

Linky is two-sided (chat + payments), so the standard verification scenario is
two app instances with two test identities on testnet. From the repo root,
`scripts/dev-two-sims.sh` boots two simulators, builds once, installs the dev
app on both against a single Metro instance, and `scripts/verify-app.sh <udid>`
checks each instance rendered via agent-device. See
[`scripts/README.md`](../../scripts/README.md) and the throwaway identities in
[`dev/test-identities/`](../../dev/test-identities/README.md).

## Routes

Expo Router, file-based, under `app/`:

```
app/
  _layout.tsx          root stack (settings/detail screens push over the tabs)
  (tabs)/
    _layout.tsx        boot gate (#14) + bottom tab bar with swipe (#16)
    index.tsx          Contacts (placeholder)
    wallet.tsx         Wallet (placeholder)
  onboarding/
    _layout.tsx        first-run flow (#17)
    index.tsx          language choice (cs/en)
    create.tsx         create account (core createIdentitySession)
    restore.tsx        "I'm returning" stub — filled by #18
    profile.tsx        name + deterministic avatar (+ optional photo)
    backup.tsx         20-word backup prompt (never blocks app entry)
  settings/            settings stack (#16), via the header menu
  contact/[id].tsx     chat/[id].tsx  wallet/token/[id].tsx   detail placeholders
```

Feature screens land per the feature map; remaining placeholders are
Contacts/Wallet surfaces and the detail routes.
