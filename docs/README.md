# Linky Next Docs

This folder tracks product and engineering planning for the Linky rewrite.

## Current Documents

- [Initial rewrite discussion](./initial-rewrite-discussion.md) - summary of the first planning conversation and agreed rewrite constraints.
- [Feature map](./feature-map/index.md) - concise inventory of what Linky does before rewrite spec and roadmap planning.
- [Rewrite tech spec](./rewrite-spec.md) - engineering spec for the rewrite: stack, monorepo layout, core architecture, environments, dev workflow.
- [Planning handout prompt](./planning-handout-prompt.md) - reusable prompt for brainstorming `rewrite-spec.md` and `feature-map.md`.

## Planning Workflow

Roadmap execution should be managed as GitHub-style issues and shown on the GitHub Project board:

- Repository: https://github.com/kaladivo/linky-next
- Project board: https://github.com/users/kaladivo/projects/4/views/1

Markdown docs in this folder should define durable context, decisions, and constraints. Implementation work should be broken into GitHub issues with clear acceptance criteria.
